# 제품 요구사항 정의서 (PRD) - 맥주 트래커

## 1. 개요

* **제품명:** 맥주 트래커 (Beer Tracker)
* **제품 목표:** 사용자가 자신의 맥주 소비량을 간편하게 기록하고, 월별/연도별 통계를 시각적으로 확인하여 음주 습관을 관리할 수 있도록 돕는 모바일 웹 애플리케이션입니다.
* **대상 사용자:** 자신의 맥주 소비량을 체계적으로 기록하고 관리하고자 하는 개인 사용자.

## 2. 핵심 기능

### 2.1. 페이지 구성

* **0. 로그인 페이지 (Login Page)**
    * **목표:** 사용자 인증을 통해 개인화된 데이터에 접근할 수 있도록 합니다.
    * **UI 요소:**
        * "맥주 트래커" 로고 및 설명 텍스트 ("맥주 소비량을 기록하고 관리하세요")
        * ID 입력 필드 (라벨: "아이디")
        * 비밀번호 입력 필드 (라벨: "비밀번호")
        * "자동 로그인" 체크박스
        * "로그인" 버튼
        * "회원가입" 버튼
        * 데모용 로그인 정보 안내 텍스트 ("아이디 3자 이상, 비밀번호 4자 이상 입력")
    * **기능:**
        * 사용자 ID와 비밀번호를 입력하여 로그인합니다.
        * "자동 로그인" 체크 시, 다음 접속 시 자동으로 로그인됩니다. (세션 또는 로컬 스토리지 활용 추측)
        * "회원가입" 버튼 클릭 시 회원가입 페이지로 이동합니다. (현재 PRD에는 회원가입 페이지에 대한 상세 정의는 포함되어 있지 않음)
        * ID 또는 비밀번호가 유효하지 않을 경우 오류 메시지를 표시합니다.
    * **참고:** `0.png` 이미지 참조

* **1. 맥주 리스트 페이지 (Beer List Page)**
    * **목표:** 사용자가 즐겨 마시는 맥주 정보를 등록, 관리하고, 음주 기록 시 선택할 수 있도록 합니다.
    * **UI 요소:**
        * 상단 "맥주 리스트" 제목
        * 우측 상단 "추가" 버튼
        * 등록된 맥주 목록 (맥주명, 제조사, 용량, 알코올 도수 표시)
        * 각 맥주 항목별 편집/삭제 아이콘
        * 각 맥주 항목별 정렬 순서 변경을 위한 핸들 아이콘 (점 3개 아이콘)
    * **기능:**
        * "추가" 버튼 클릭 시 새로운 맥주를 등록할 수 있는 페이지/모달로 이동합니다. (맥주명, 제조사, 용량, 알코올 도수 입력 필드 포함)
        * 등록된 맥주 정보를 수정하거나 삭제할 수 있습니다.
        * 등록된 맥주 항목의 정렬 순서를 드래그 앤 드롭 또는 화살표 버튼 등으로 변경할 수 있습니다.
        * 하단 네비게이션 바에서 "맥주 리스트" 탭이 활성화됩니다.
    * **참고:** `1.png` 이미지 참조

* **2. 맥주 소비량 등록 페이지 (Beer Consumption Registration Page)**
    * **목표:** 특정 날짜에 마신 맥주 종류와 수량을 기록합니다.
    * **UI 요소:**
        * 상단 "음주 기록" 제목
        * "날짜 선택" 필드 (기본값: 현재 날짜, 캘린더 아이콘 클릭 시 날짜 선택 UI 표시)
        * 등록된 맥주 종류별 수량 조절 섹션 (맥주명, 용량, 알코올 도수, 마이너스 버튼, 현재 수량, 플러스 버튼)
        * "선택한 날짜의 음주량" 요약 정보 (총 수량, 총 용량, 평균 알코올 도수)
        * "기록 저장" 버튼
    * **기능:**
        * 날짜 선택기를 통해 음주 기록을 남길 날짜를 선택합니다.
        * 각 맥주 종류별로 마신 수량을 +/- 버튼을 통해 조절합니다.
        * 수량 조절 시 "선택한 날짜의 음주량" 요약 정보가 실시간으로 업데이트됩니다.
        * "기록 저장" 버튼 클릭 시 해당 날짜의 음주 기록이 저장됩니다.
        * 이미 기록된 날짜의 경우, 해당 기록을 불러와 수정할 수 있도록 합니다.
        * 하단 네비게이션 바에서 "음주 기록" 탭이 활성화됩니다.
    * **참고:** `2.png` 이미지 참조

* **3. 통계 페이지 (Statistics Page)**
    * **목표:** 사용자의 맥주 소비 기록을 월별, 연도별로 시각화하여 보여줍니다.
    * **UI 요소:**
        * 상단 "통계" 제목 및 "월별/연도별" 필터 드롭다운 메뉴
        * **월별 캘린더 뷰:**
            * 이전/다음 월 이동 버튼
            * 현재 표시 월/연도 (예: "2025년 7월")
            * 날짜별 박스 (음주 기록이 있는 날짜는 수량이 표시되며, 수량이 많을수록 색이 진해집니다. 예: `3.png`의 27일, 28일)
            * 하단 "색이 진할수록 더 많이 마셨습니다" 설명 문구
        * **월별/연도별 요약 섹션:**
            * 총 음주 일수
            * 총 소비량 (잔)
            * 총 음주량 (L 또는 ml)
            * 최대 음주량 (잔, 월별/연도별)
            * 총 알코올량 (ml)
            * 평균 (음주일 기준, 잔)
        * **선호 맥주 섹션:**
            * 자주 마신 맥주 목록 (맥주명, 용량, 알코올 도수, 총 마신 잔 수, 총 마신 용량)
    * **기능:**
        * "월별/연도별" 드롭다운을 통해 통계 조회 단위를 변경할 수 있습니다.
        * **월별 통계:**
            * 선택된 월의 캘린더에 음주 기록이 있는 날짜와 수량을 표시합니다.
            * 해당 월의 총 음주 일수, 총 소비량, 총 음주량, 최대 음주량, 총 알코올량, 평균 음주량을 계산하여 보여줍니다.
            * 해당 월에 가장 많이 마신 맥주 순위를 보여줍니다.
        * **연도별 통계:**
            * 각 월별 총 소비량과 총 음주량을 보여줍니다.
            * 해당 연도의 총 소비량, 총 음주량, 총 알코올량, 월 평균 음주량을 계산하여 보여줍니다.
            * 해당 연도에 가장 많이 마신 맥주 순위를 보여줍니다.
        * 하단 네비게이션 바에서 "통계" 탭이 활성화됩니다.
    * **참고:** `3.png` (월별 통계), `4.png` (연도별 통계) 이미지 참조

### 2.2. 공통 기능 및 UI

* **모바일용 웹앱:** 반응형 웹 디자인을 적용하여 모바일 환경에 최적화된 사용자 경험을 제공합니다.
* **하단 네비게이션 바:**
    * "맥주 리스트" (Beer List)
    * "음주 기록" (Drink Log)
    * "통계" (Statistics)
    * 현재 활성화된 탭은 시각적으로 강조됩니다.
* **상단 헤더:**
    * "맥주 트래커" 앱 이름
    * 로그아웃 버튼 (우측 상단)

## 3. 기술 요구사항

* **데이터베이스:**
    * 사용자 계정 정보 (ID, 비밀번호, 자동 로그인 여부 등) 저장
    * 맥주 정보 (맥주명, 용량, 알코올 도수, 제조사, 정렬 순서 등) 저장
    * 음주 기록 (날짜, 맥주 ID, 수량 등) 저장
    * 관계형 또는 NoSQL 데이터베이스 고려 (Vercel 환경 고려 시 PostgreSQL, MongoDB Atlas 등 검토)
* **프론트엔드:**
    * 모바일 웹앱 개발에 적합한 JavaScript 프레임워크/라이브러리 (예: React, Vue.js, Svelte 등)
    * 날짜 선택, 캘린더 뷰, 차트 등 UI/UX 구현 라이브러리 활용
* **백엔드 (API):**
    * 데이터베이스와 연동하여 데이터 CRUD (Create, Read, Update, Delete) 작업을 수행하는 API 개발
    * 사용자 인증 및 권한 부여
* **배포 환경:**
    * **Vercel 배포:** Vercel 환경에 배포 가능하도록 코드 구성 및 설정 (환경 변수 관리 등)
    * **Vercel DB 연동:** Vercel 플랫폼에서 제공하거나 연동 가능한 데이터베이스 서비스 (예: Vercel Postgres, Supabase, PlanetScale 등)를 활용하여 데이터베이스 설정 및 연동
